# Migration: v0.x.y -> v1.x.y

## Summary

v1 updates the `caffeineai-authorization` mops package to `1.0.0` and adds support for Internet Identity attribute bundles (verified email). The primary breaking change is that `MixinAuthorization` now requires a second argument. This release also requires `@caffeineai/core-infrastructure ^1.0.0` on the frontend — see the [core-infrastructure v0.x.y->v1.x.y migration](../../extension-core-infrastructure/migration/v0.x.y-to-v1.x.y.md) for the corresponding frontend changes.

## Breaking changes

### 1. `mops.toml` version bump

```diff
-caffeineai-authorization = "~0.2.0"
+caffeineai-authorization = "~1.0.0"
```

The package now depends on `identity-attributes` and requires `core = "2.5.0"` and `moc 1.6.0`. Update your `mops.toml` toolchain if needed:

```toml
[toolchain]
moc = "1.6.0"

[dependencies]
core = "2.5.0"
caffeineai-authorization = "~1.0.0"
```

### 2. `MixinAuthorization` now takes a second argument

The mixin signature changed to accept an optional callback:

```
mixin (accessControlState, onAttributesVerified : ?((Principal, { name : ?Text; email : ?Text; sso : ?Text }) -> ()))
```

Every existing `include MixinAuthorization(accessControlState)` must be updated to pass `null` as the second argument:

```diff
-include MixinAuthorization(accessControlState);
+include MixinAuthorization(accessControlState, null);
```

To capture the verified email at sign-in, pass a callback instead of `null` — see the **Email Attributes** section of the [authorization SKILL.md](../SKILL.md#email-attributes) for the full pattern.

### 3. Frontend requires `@caffeineai/core-infrastructure ^1.0.0`

The II attribute flow (nonce, `requestAttributes`, `_internet_identity_sign_in_finish`) is handled by `@caffeineai/core-infrastructure`. Apps using `caffeineai-authorization ^1.0.0` on the backend must also run `@caffeineai/core-infrastructure ^1.0.0` on the frontend.

## Upgrade checklist

- [ ] Bump `caffeineai-authorization` to `~1.0.0` in `mops.toml`
- [ ] Bump `core` to `"2.5.0"` and `moc` to `"1.6.0"` in `mops.toml`
- [ ] Add `null` as the second argument to every `include MixinAuthorization(accessControlState)` call in `main.mo`
- [ ] Ensure the frontend runs `@caffeineai/core-infrastructure ^1.0.0` (see core-infrastructure migration)
- [ ] Run `mops install` and verify the backend compiles without errors
